1.2 Entity: Cours
Table: cours

Field	Java Type	DB Column	Notes
id	Long	id (PK, auto-increment)	Generated — never send on POST
title	String	title	Required for display
description	String	description (varchar 1000)	
content	String	content (varchar 5000)	Long-form course content
contenus	List<ContenuPedagogique>	— (mapped by child)	Included in GET responses
Relationships:

@OneToMany(mappedBy = "cours", cascade = CascadeType.ALL, orphanRemoval = true) → contenus
@JsonManagedReference on contenus — this side IS serialized in JSON responses
Fetch: LAZY (default for @OneToMany), but the service uses JOIN FETCH queries so contenus are always loaded for GET endpoints
Persistence behavior:

CascadeType.ALL means if you save a Cours with contenus populated, the children are also persisted
orphanRemoval = true means removing a child from the list and saving the parent will DELETE the child row
However, the updateCours service method only updates title, description, content — it does NOT touch contenus. To add/remove contenus, use the dedicated ContenuPedagogique endpoints.

3. DATA FLOW AND PERSISTENCE
3.1 Nested Object Saving: Cours → ContenuPedagogique
The Cours entity has cascade = CascadeType.ALL on contenus. In theory, you could POST a Cours with nested contenus and they'd all be saved. However, the current createCours service simply calls coursRepository.save(cours), so this works only if the incoming JSON includes contenus with each child's cours back-reference properly set.

Recommended approach (simpler, more reliable):

POST the Cours first (without contenus) → get back the id
POST each ContenuPedagogique separately with cours: { id: <coursId> }
The same pattern applies to Quiz → QuestionQuiz.

3.2 Update Behavior
All update* methods use a field-by-field copy pattern. They:

Load the existing entity from DB
Copy only scalar fields from the request body
Save the existing entity
This means:

updateCours updates title, description, content — NOT contenus
updateQuiz updates title, description, level, dateStart, dateEnd, status, courseId, xpReward — NOT questions
To modify children, use the child's own endpoints
3.3 Delete Behavior
Deleting a Cours cascades to delete all its ContenuPedagogique children
Deleting a Quiz cascades to delete all its QuestionQuiz children (and their question_options rows)
All other entities are standalone — deleting them has no cascade effects
3.4 Pitfalls That Cause Missing Data
Pitfall	Explanation
cours field missing in ContenuPedagogique JSON response	@JsonBackReference suppresses it. The frontend must track the parent coursId separately.
quiz field missing in QuestionQuiz JSON response	Same reason — @JsonBackReference.
Sending id on POST	The id is auto-generated. If you send an id, it will be ignored on insert (GenerationType.IDENTITY).
Empty contenus array on Cours creation	Normal — contenus should be added separately via their own endpoint.
Enum case sensitivity	Must be exact uppercase: "BEGINNER" not "beginner". A wrong case causes a 400 error.
LocalDate format	Must be "YYYY-MM-DD" (e.g. "2025-06-15").
LocalDateTime format	Must be "YYYY-MM-DDTHH:mm:ss" (e.g. "2025-06-15T14:30:00").
options on QuestionQuiz	Sent as a plain JSON array ["A", "B", "C", "D"]. Stored in a separate table automatically.

4.3 Example Angular Service (Cours)
typescript
@Injectable({ providedIn: 'root' })
export class CoursService {
  private baseUrl = `${environment.apiBaseUrl}/cours`;
 
  constructor(private http: HttpClient) {}
 
  getAll(): Observable<Cours[]> {
    return this.http.get<Cours[]>(`${this.baseUrl}/get-all-cours`);
  }
 
  getById(id: number): Observable<Cours> {
    return this.http.get<Cours>(`${this.baseUrl}/get-cours-by-id/${id}`);
  }
 
  create(cours: Cours): Observable<Cours> {
    return this.http.post<Cours>(`${this.baseUrl}/create-cours`, cours);
  }
 
  update(id: number, cours: Cours): Observable<Cours> {
    return this.http.put<Cours>(`${this.baseUrl}/update-cours/${id}`, cours);
  }
 
  delete(id: number): Observable<void> {
    return this.http.delete<void>(`${this.baseUrl}/delete-cours/${id}`);
  }
}

4.5 How Users Should Interact (Role-Based UI Logic)
Role	Can Do	Cannot Do
User (Student)	GET all courses, quizzes, sessions, certifications, quiz cards, categories, practice items. View course details with contenus. View quiz with questions.	POST, PUT, DELETE on any entity.
Tutor	Everything User can do + POST/PUT/DELETE on: Cours, ContenuPedagogique, Quiz, QuestionQuiz, PracticeItem, QuizCard, QuizCategory, Certification.	POST/PUT/DELETE on Sessions (admin only).
Admin	Everything Tutor can do + POST/PUT/DELETE on Sessions.	—
Frontend enforcement: Since the backend has no auth, the Angular app must:

Hide create/edit/delete buttons based on the logged-in user's role
Optionally guard routes with Angular route guards
A future API gateway or Spring Security layer should enforce this server-side
5. POTENTIAL BACKEND GOTCHAS
5.1 Lazy Loading Exceptions
Cours.contenus and Quiz.questions are LAZY-loaded
The service layer uses JOIN FETCH queries for GET endpoints, so these are safe
However, if you access contenus or questions outside a transaction (e.g., in a custom endpoint without JOIN FETCH), you'll get a LazyInitializationException
The default_batch_fetch_size=20 in config mitigates N+1 for any remaining lazy scenarios
5.2 Infinite Recursion Prevention
Cours ↔ ContenuPedagogique: handled by @JsonManagedReference / @JsonBackReference
Quiz ↔ QuestionQuiz: handled by @JsonManagedReference / @JsonBackReference
Consequence: The child's parent reference (cours, quiz) is never present in JSON GET responses. The frontend must track parent IDs separately.
5.3 Fields Ignored During Deserialization
@JsonBackReference fields (ContenuPedagogique.cours, QuestionQuiz.quiz) ARE accepted on deserialization (POST/PUT). You CAN send "cours": {"id": 1} and it will be deserialized and persisted. The annotation only suppresses serialization (output).
spring.jackson.deserialization.fail-on-unknown-properties=false — unknown fields in request JSON are silently ignored, not rejected.
5.4 No CORS Configuration
There is no CORS config. Angular running on localhost:4200 will get CORS errors when calling localhost:8086.
Fix options:
Add @CrossOrigin(origins = "http://localhost:4200") to each controller

5.5 ContenuPedagogique Primary Key Name
The PK field is idContent, not id. The JSON key in responses will be "idContent". The frontend must use contenu.idContent to reference it, not contenu.id.
5.6 Quiz ↔ Course Link
Quiz.courseId is a plain Long, not a JPA relationship. There is no DB-level foreign key constraint. The frontend should validate that the courseId references a valid course before sending.
5.7 Error Responses
All "not found" scenarios throw RuntimeException with a message like "Cours not found with id: 99". Spring Boot will return this as a 500 Internal Server Error with the message in the response body. There is no custom exception handler returning 404. The frontend should handle 500 responses for "not found" scenarios.
